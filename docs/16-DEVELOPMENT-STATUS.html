cron روزانه: compute برای مخاطبانی که lifecycle=customer/vip/at_risk؛
GET /v1/reports/at-risk-contacts؛
notification به اپراتور assign‌شده هنگام score زیر threshold؛
مرتبط با FL-47 (Scoring) و FL-46 (Lifecycle)

معماری کلی CRM ردیابی مادام‌العمر:
Widget cookie (cbx_vid, 1yr) → visitor_events (partitioned, unlimited) → identify() call → visitor_identities (merge ناشناس↔شناخته‌شده) → contact profile (custom attrs + lifecycle + score + health) → Journey UI (نمایش کامل سابقه از روز اول).
Privacy: داده‌ها workspace-scoped هستند. retention policy per-plan. مشتری می‌تواند با cbx('forget') درخواست حذف کند (GDPR/حریم خصوصی).

## الگوی استاندارد پیاده‌سازی Feature

### مرحله A — DB (اگر نیاز به جدول/فیلد جدید است)

3. RLS policy اضافه کنید اگر جدول workspace-scoped است
4. seed را به‌روز کنید اگر داده پیش‌فرض نیاز است

### مرحله B — API (Fastify route)
```typescript
// الگوی route فایل
import { FastifyPluginAsync } from 'fastify'
import { requireAuth, requireRole } from '../../auth/middleware.js'

const myRoute: FastifyPluginAsync = async (app) => {
  app.get('/v1/my-resource', { preHandler: [requireAuth] }, async (req, reply) => {
    const { workspaceId } = req.user  // همیشه workspace isolation
    // ...
  })
}
```
- route را در `apps/api/src/app.ts` register کنید
- error format: `{ error: { code, message, details? } }`
- همیشه workspaceId را از `req.user` بگیرید، نه از body/param

### مرحله C — Real-time (اگر نیاز است)
```typescript
// emit در سمت server
io.to(`workspace:${workspaceId}`).emit('resource:event', payload)

```

### مرحله D — Dashboard UI (Next.js)
1. صفحه: `apps/dashboard/src/app/(dashboard)/my-page/page.tsx`
2. کامپوننت‌ها: `apps/dashboard/src/components/my-feature/`
3. API calls: از `apps/dashboard/src/lib/api.ts` (fetch wrapper موجود) استفاده کنید
4. RTL: Tailwind + `dir="rtl"` — از `text-right` و `mr-/ml-` آگاه باشید

### مرحله E — تست
1. unit test: `apps/api/src/__tests__/my-feature.test.ts` (vitest + supertest)
2. حداقل ۳ test: happy path، not found، unauthorized
3. اجرا: `pnpm test`

۴ — قواعد مهم (Rules Agent باید رعایت کند)

قانونتوضیح
Workspace Isolation هر query باید WHERE workspace_id = $workspaceId داشته باشد. بدون این، cross-tenant data leak رخ می‌دهد.
Auth در هر route preHandler: [requireAuth] را فراموش نکنید. برای admin-only: requireRole('admin').
Migration immutable فایل migration موجود را ویرایش نکنید. همیشه migration جدید بسازید.
Type safety تایپ‌های shared را در packages/shared بسازید، نه در app اختصاصی.
Error format همیشه از reply.status(4xx).send({ error: { code, message } }) استفاده کنید.
RTL in UI کامپوننت‌های داشبورد RTL هستند. از gap-x به‌جای space-x در flex استفاده کنید.
Plan limits اگر فیچر جدید محدودیت پلن دارد، به apps/api/src/billing/plan-limits.ts اضافه کنید.
env.example هر متغیر محیطی جدید را با کامنت توضیحی به .env.example اضافه کنید.
